# Realsense Camera Usage

## What it does

This skill documents and executes the standard RealSense camera workflow for ROSClaw bodies equipped with Intel RealSense D405 and/or D435i cameras:

- Enumerate active RealSense ROS2 nodes and topics.
- Check camera online status (`get_camera_status`).
- Capture a synchronized RGB-D frame (`capture_rgbd`).
- Validate depth frame health (`check_depth_validity`).
- Return structured camera status, frame paths, and any recoverable errors.

It is intended as a reusable reference skill for perception-only bodies and as a starting point for downstream skills such as `scene_risk_scan`.

## Supported robots

- `realsense_dual` — D405 + D435i dual-camera setup.
- `realsense_d405` — single D405 camera.
- `realsense_d435i` — single D435i camera (with optional IMU).

## Required sensors

- `color_camera`
- `depth_camera`
- `imu` (D435i only)

## Required providers

- See `providers.yaml`. No external inference provider is required for basic usage; the skill only needs the local `realsense-ros-mcp` or `librealsense-mcp` camera server bound to the body.

## Safety constraints

- See `safety.yaml`.
- Default runtime mode: `sandbox_first`.
- This is a **read-only perception skill**; it does not command actuators or robot motion.
- `real_robot_guarded` is allowed because only sensor reads are performed.

## How to run

```bash
# Check camera status for the active body
rosclaw skill invoke realsense_camera_usage --body d405_lab_01 --json

# Capture RGB-D explicitly
rosclaw skill invoke realsense_camera_usage \
  --body d435i_lab_01 \
  --params '{"camera_name":"d435i","output_dir":"/tmp/realsense_usage"}' \
  --json

# Validate the package
rosclaw skill validate realsense_camera_usage --json

# Package and dry-run upload
rosclaw skill package realsense_camera_usage --output dist --json
rosclaw skill upload realsense_camera_usage --dry-run --json
```
